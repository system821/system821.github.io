:cherry_blossom:事务

  • 事务就是一个完整的业务逻辑
  • 只有DML才有事务这一说，其他语句和事务无关！
  • insert、delete、update
  • 批量的DML，同时成功，同时失败

事务是怎么做到多条DML语句同时成功，同时失败？

InnoDB: 提供一组来记录仪事务性活动的日志文件

  1. 在事务的提交过程中，每一条DML的操作都会记录到“事务性活动的日志文件”中
  2. 在事务的执行过程中，我们可以提交事务，也可以回滚事务。

提交事务？

清空事务性活动日志文件，将数据全部彻底持久化到数据库表中

提交事务标志着，事务的结束，并且是一种全部成功的结束

回滚事务？

将之前所有的DML操作全部撤销，并且清空事务性活动的日志文件

回滚事务标志着，事务的结束，并且是一种全部失败的结束

怎么提交事务？怎么回滚事务？

提交事务：commit

回滚事务：rollback

开始事务：start transaction

事务的四个性质？

ACID

  1. 原子性

    说明事务是最小的工作单元，不可再分

  2. 一致性

    确保事务是同时成功或者是同时失败

  3. 隔离性

    事务A与事务B之间具有一定的隔离

  4. 持久性

    事务最终结束的一个保障，事务提交，就相当于将没有保存到硬盘上的数据保存到硬盘上！

事务的隔离性

  1. 读未提交 read uncommitted（最低的隔离级别）

    脏读

  2. 读已提交 read committed

    解决脏读现象

    不可进行重复读取数据

    每一次得到的数据是绝对真实

  3. 可重复读(repeatable read)

    事务A开始之后，不管是多久，每一次在事务A中读取到的数据都是一直的，即使事务B将数据已经修改

    解决了不可重复读的文图

    可能会读幻影

  4. 序列化/串行化(最高的隔离级别)

    解决了所以的问题，但是不能并发

索引(index)

概述

索引是在数据库表的字段上添加的，是为了提高查询效率存在的一种机制。

一张表的一个字段可以添加一个索引，当然，多个字段联合起来也可以添加索引

索引相当于一本书的目录，是为了缩小查询范围

MySQL在查询方面主要就是两种方式：

  1. 全表扫描
  2. 根据索引检索

:strawberry:在mysql数据库中索引也是需要排序的，并且这个索引的排序和TreeSet数据结构相同，TreeSet底层是一个自平衡的二叉树！在mysql中索引是一个B-Tree数据结构

什么时候添加索引

  1. 数据量庞大
  2. 该字段经常出现咋where后面，以条件的形式存在，也就说这个字段总是被扫描
  3. 该字段很少的DML操作
